# 📌 プロジェクト全体ロードマップ

## 🎯 プロジェクト全体目的
フリーランスとしての金銭管理・証憑管理を、**低コスト（ツール課金なし）**かつ**継続可能**な運用で実現する。会計申告の自動化ではなく、**自分の目でお金の流れを把握**し、将来的に必要なら手動で確定申告に使えるデータを準備できる状態を保つ。

## 🗓 実装スケジュール

### Week 1-6: Phase 1 基盤構築 & MVP
- **基盤:** Docker環境構築、Supabase初期設定、DDL投入。
- **インポート:** Python Importer実装（1種類のカードCSV）、fingerprint生成、重複検出。
- **UI:** Transactions一覧、Triage Queue、月次集計ダッシュボード。

### Month 3-4: Phase 2 表記ゆれ＋効率化
- **マスタ:** vendors/vendor_aliases/expense_categoriesテーブル追加。
- **効率化:** Smart Suggestions（過去判定の再利用）、Batch Actions（一括ルール）。
- **拡張:** 複数入力経路（口座、d払い、現金）への対応。

### 必要に応じて: Phase 3 証憑管理強化
- **OCR:** 画像からの金額・日付自動抽出。
- **出力:** 汎用CSV/Excelエクスポート機能。
- **自動化:** Gmailからの領収書PDF自動取り込み。

### 外出先で使いたくなったら: Phase 4 クラウド化
- **デプロイ:** Vercel/Supabase Cloudへの移行。
- **認証:** Supabase Authによるログイン管理。
- **モバイル:** レスポンシブUI、モバイルからのレシート撮影・アップロード。

## 🎯 成功の定義

| フェーズ | 成功基準 |
| :--- | :--- |
| **Phase 1** | 1ヶ月分の明細を取り込み、未判定をゼロにできる。ローカルで快適に動作する。 |
| **Phase 2** | 表記ゆれが吸収され、判定作業時間が50%削減される。 |
| **Phase 3** | レシートアップロードで自動抽出ができ、必要に応じてデータ出力ができる。 |
| **Phase 4** | 外出先からスマホで確認・入力ができ、安全にクラウド管理されている。 |
