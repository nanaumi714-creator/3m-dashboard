# 今後の課題 (Roadmap)
**最終更新日: 2026年02月01日**

レシート管理および資産管理機能の強化に向けた今後のタスクリストです。

## 1. 支払方法の機能拡張
- [ ] **支払方法のカスタム追加**: ユーザーが独自の支払方法（特定のクレジットカード名など）を自由に追加・管理できるようにする。
- [ ] **引き落としタイミングの設定**:
    - 支払方法ごとに「残高引き落とし（即時）」か「翌月引き落とし（後払い）」かを選択可能にする。
    - **デフォルト設定**:
        - 残高引き落とし: 現金、銀行振込、QR決済、電子マネー
        - 翌月引き落とし: カード

## 2. 残高および資産管理機能
- [ ] **現在残高の登録**:
    - 初期状態の残金を登録できるようにする。
    - 残金は以下の「種別」ごとに分類して管理。
    - **種別一覧**: 「現金」「QR決済」「銀行」「電子マネー」
- [ ] **資産の管理場所（管理ソース）の管理**:
    - 実際に残金を記録する「場所」を追加・編集・削除できるようにする。
    - 外部サービスとの直接的な連携（API連携等）は行わず、あくまで本システム内での「管理上の場所」として定義する。
    - 例: 財布、三菱UFJ銀行、PayPay、Suicaなど。
- [ ] **資金移動（振替）機能**:
    - 登録された管理場所間での資金移動を記録できるようにする（例: 銀行から引き出して現金（財布）へ移動するなど）。

## 3. UI/UX 改善
- [ ] **カテゴリ・支払方法の自動推定精度の向上**: LLMのプロンプト調整および過去の履歴に基づく学習。
- [ ] **一括編集機能**: 未処理一覧からの効率的なデータ入力。
