# Phase 4: クラウド化（外出先からの入力・確認・課金許容）

## 要件

### 目的
- ローカル環境からクラウド環境への移行
- **外出先からデータ入力・確認**ができるようにする
- モバイルからのレシート登録に対応
- **課金許容**（Vercel、Supabase Cloud等）

### スコープ

#### 1. クラウドデプロイ
- Vercel等へのデプロイ
- Supabase Cloud移行
- 認証機能（Supabase Auth - メール/パスワード）
- HTTPS通信
- 環境変数の適切な管理
- CI/CDパイプライン（GitHub Actions推奨）

#### 2. 基本的なユーザー管理
- 単一ユーザー運用（自分専用）
- ログイン/ログアウト機能
- パスワード変更機能
- パスワードリセット（メール経由）
- Row Level Security (RLS) 有効化
- セッション管理

#### 3. モバイル対応UI
- レスポンシブデザイン対応（全画面）
- モバイルブラウザからのレシート撮影→アップロード
  - カメラ起動
  - プレビュー表示
  - 即座にアップロード
- モバイルからのTransactions一覧閲覧
  - スワイプジェスチャー
  - プルトゥリフレッシュ
- モバイルからのTriage Queue操作
  - タップで判定
  - スワイプで次へ
- モバイル専用簡易入力フォーム

#### 4. 外出先での入力
- Web UIからの手動取引入力
  - 入力フォームがモバイルフレンドリー
  - 日付ピッカー最適化
  - 金額入力の数値キーボード
- モバイルブラウザからのレシート即時アップロード
- 簡易メモ機能（後で整理）
  - 下書き保存
  - 一時保存リスト
- オフライン対応（任意）
  - IndexedDBでローカル保存
  - オンライン復帰時に同期

#### 5. データ移行
- ローカル環境のデータをクラウドに移行
  - 移行スクリプト
  - データ整合性チェック
  - ロールバック手順
- 移行後もローカル環境が使用可能（開発用）
- データ同期機能（任意）

#### 6. セキュリティ強化
- Row Level Security (RLS) ポリシー設定
  - ユーザー別データ分離
  - テーブル単位のアクセス制御
- API Rate Limiting
- CORS設定
- CSP（Content Security Policy）
- 不正アクセス検知（任意）
