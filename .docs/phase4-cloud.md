# Phase 4: クラウド化（外出先からの入力・確認）

## 1. 目的
- ローカル環境からクラウド環境への移行。
- **外出先からデータ入力・確認**ができるようにする。
- モバイルからのレシート登録に対応。

## 2. スコープ

### 2.1 クラウドデプロイ
- Vercel等へのデプロイ。
- Supabase Cloud移行。
- 認証機能（Supabase Auth - メール/パスワード）およびHTTPS通信。

### 2.2 基本的なユーザー管理
- 単一ユーザー運用（自分専用）。
- ログイン/ログアウト、パスワード変更機能。

### 2.3 モバイル対応UI
- レスポンシブデザイン対応。
- モバイルブラウザからのレシート撮影→アップロード。
- モバイルからのTransactions一覧閲覧、Triage Queue操作。

### 2.4 外出先での入力
- Web UIからの手動取引入力。
- モバイルブラウザからのレシート即時アップロード。
- 簡易メモ機能（後で整理）。

## 3. 受け入れ基準

### 3.1 クラウド環境 & 認証
- [ ] Vercel/Supabase Cloudにデプロイされ、HTTPSで外部からアクセスできる。
- [ ] 認証機能（ログイン/ログアウト/パスワードリセット）が動作する。
- [ ] Row Level Security (RLS) により未認証アクセスの制限ができる。

### 3.2 モバイル対応 & 外出先入力
- [ ] スマートフォンブラウザでレスポンシブに表示され、カメラ起動・画像アップロードができる。
- [ ] Web UIからモバイルフレンドリーなフォームで手動取引入力ができる。

### 3.3 データ移行
- [ ] ローカル環境のデータをクラウドに移行できる。
- [ ] 移行後もローカル環境を開発用として使用可能。
