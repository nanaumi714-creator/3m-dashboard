# GUI入力機能の洗い出し（CSVインポート・手入力・編集）

## 目的
コマンドベースの入力から脱却し、GUI上でのCSVインポート・手入力・編集を可能にするための必要機能を整理する。

## CSVインポート（GUI）
### 画面・フロー
- アップロード → プレビュー → 取り込み確定の3ステップ。
- 取り込み結果（件数・重複・エラー）を表示。
- 失敗時のエラー表示と再試行導線。

### フォーマット確認（必須）
- `docs/csv_spec.md` の仕様に一致するかを検証。
- 必須列の存在、列名の一致、型の妥当性（数値/日付/文字列）をチェック。
- 失敗時は具体的な列・理由を提示し、取り込みをブロック。

### 重複/冪等性
- チェックサム/指紋による重複検出をGUIからも実行。
- 重複時は「再取り込みなし」で終了できるUXにする。

## 手入力
- 必須項目: `occurred_on`, `amount_yen`（負数）, `payment_method_id`, `description`。
- 任意項目: `vendor_raw` or `vendor_id`, `category_id`, `memo`。
- 入力時バリデーション（JPYのみ、負数、必須項目）。
- 失敗時のエラー表示と保存中のローディング状態。

## 編集
- 取引詳細画面からの編集導線。
- 編集対象: 日付、金額、説明、ベンダー、カテゴリー、支払方法。
- `transaction_business_info`（事業判定・比率・監査メモ）も編集可能にする。

## レシート（補助機能）
- 取引に紐づくレシート画像のアップロード。
- OCR実行と結果の表示・検索対象化。

## UI設計方針
- すべてカードベース（白背景・角丸・影・境界線）。
- Tailwindユーティリティのみ使用。
- ローディング・エラーは明示的に表示。
